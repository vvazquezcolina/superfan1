**1. Create `src/lib/breadcrumbs.ts`** -- Utility for generating breadcrumb data and JSON-LD:

```typescript
import type { Locale } from '@/lib/content/schemas'

const SITE_URL = 'https://www.superfaninfo.com'

export interface BreadcrumbItem {
  label: string
  href: string
}

// Map from filesystem path segments to dictionary keys
const SEGMENT_TO_DICT_KEY: Record<string, string> = {
  ciudades: 'cities',
  estadios: 'stadiums',
  equipos: 'teams',
  viajes: 'travel',
  herramientas: 'tools',
}

// Map from filesystem segments to English URL segments
const SEGMENT_TO_EN: Record<string, string> = {
  ciudades: 'cities',
  estadios: 'stadiums',
  equipos: 'teams',
  viajes: 'travel',
  herramientas: 'tools',
}
```

Export function `generateBreadcrumbs`:
```typescript
export function generateBreadcrumbs(
  path: string,            // e.g., '/es/ciudades/ciudad-de-mexico'
  lang: Locale,
  breadcrumbLabels: Record<string, string>,  // dict.breadcrumbs
  entityName?: string,     // e.g., 'Ciudad de Mexico' for the leaf
): BreadcrumbItem[]
```

Logic:
- Split path by `/`, filter empty strings, remove the lang prefix
- First item is always Home: `{ label: breadcrumbLabels.home, href: '/${lang}' }`
- For each remaining segment (except the last if entityName provided):
  - Look up the segment in `SEGMENT_TO_DICT_KEY` to get the breadcrumb dictionary key
  - Use `breadcrumbLabels[dictKey]` as the label
  - Build href using the actual path segments (respecting locale -- Spanish filesystem paths for /es/, English URL paths for /en/)
- If `entityName` is provided, the last item uses `entityName` as label with no link (current page)
- If no entityName, the last segment gets its label from the dictionary

Example output for `/es/ciudades/ciudad-de-mexico` with entityName "Ciudad de Mexico":
```
[
  { label: "Inicio", href: "/es" },
  { label: "Ciudades sede", href: "/es/ciudades" },
  { label: "Ciudad de Mexico", href: "/es/ciudades/ciudad-de-mexico" }
]
```

Export function `buildBreadcrumbJsonLd`:
```typescript
export function buildBreadcrumbJsonLd(breadcrumbs: BreadcrumbItem[]): object
```

Returns a Schema.org BreadcrumbList object:
```json
{
  "@context": "https://schema.org",
  "@type": "BreadcrumbList",
  "itemListElement": [
    {
      "@type": "ListItem",
      "position": 1,
      "name": "Inicio",
      "item": "https://www.superfaninfo.com/es"
    },
    {
      "@type": "ListItem",
      "position": 2,
      "name": "Ciudades sede",
      "item": "https://www.superfaninfo.com/es/ciudades"
    },
    {
      "@type": "ListItem",
      "position": 3,
      "name": "Ciudad de Mexico"
    }
  ]
}
```

Note: The last item in the list does NOT include `item` (URL) per Schema.org convention -- it represents the current page.
Each `item` URL must be absolute (prefixed with SITE_URL).

**Approach:** Breadcrumbs are rendered at the PAGE level (not layout) because:
1. Server-rendered breadcrumbs are in the initial HTML (better for SEO)
2. Each page knows its entity name (city name, stadium name, etc.)
3. JSON-LD must be page-specific anyway
4. Layout.tsx Server Components cannot access the current URL path

IMPORTANT: Do NOT add breadcrumbs to layout.tsx. Each page renders its own.
